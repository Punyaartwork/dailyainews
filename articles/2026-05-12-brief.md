### 2. ผู้สร้าง curl ทดสอบ Claude Mythos: ได้ช่องโหว่ระดับต่ำเพียง 1 จุด — [Blognone](https://www.blognone.com/node/150501)

Daniel Stenberg ผู้สร้างเครื่องมือ network `curl` ที่ใช้งานในแทบทุก device บนโลก เปิดเผยผลการทดสอบที่ได้รับเชิญจาก Linux Foundation ให้เข้าร่วม Anthropic รัน Claude Mythos โมเดลล่าสุดและทรงพลังที่สุดของบริษัทบนโค้ด 176,000 บรรทัดของ curl และรายงานพบช่องโหว่เพียง 1 จุดในระดับ low severity Stenberg ระบุว่า curl ผ่านกระบวนการ code review หลายชั้นอยู่แล้ว ทั้ง CodeQL, Coverity, และ OSS-Fuzz ทำให้ช่องโหว่ที่เหลืออยู่น้อยมาก และ Mythos ไม่ได้ทำผลงานต่างจากโมเดลอื่นอย่างมีนัยสำคัญ

ผลนี้เป็นตัวอย่างชั้นดีของ "diminishing returns in AI security": AI ได้เปรียบชัดเจนกับ legacy codebase ที่ขาดการดูแล แต่บน codebase ที่ผ่าน automated security tooling ซ้ำแล้วซ้ำเล่า ช่องว่างจะแคบลงมาก ผู้เชี่ยวชาญด้านความปลอดภัยตั้งข้อสังเกตว่าผลลัพธ์นี้ไม่ใช่ความพ่ายแพ้ของ Mythos แต่เป็นการยืนยันว่า curl มีวุฒิภาวะด้าน security สูงมาก และความน่าสนใจจริงๆ คือ Anthropic เลือก red-team model ของตัวเองด้วยโปรเจกต์ real-world ซึ่งเป็นแนวทาง responsible AI development ที่ควรกลายเป็น industry norm สำหรับนักพัฒนา นัยในทางปฏิบัติชัดเจน: ลงทุน CodeQL, Coverity, และ OSS-Fuzz ก่อน แล้วค่อย layer AI security scan ทับ — ไม่ใช่ทางกลับกัน AI จะ contribute ได้มากที่สุดบน codebase ที่ยังมี low-hanging bug อยู่
